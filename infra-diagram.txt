  STARTUP ORDER
  ─────────────
  [1] scylla-cdc-quickstart    (seed — wait for UN state before proceeding)
  [2] scylla-cdc-quickstart-2  (+30s after seed is UN)
  [3] scylla-cdc-quickstart-3  (+30s after node 2 is UN)
  [4] scylla-cdc-broker        (KRaft init with generated cluster ID)
  [5] scylla-cdc-schema-registry
  [6] scylla-cdc-connect       (Datagen installed at boot — community only)
  [7] scylla-cdc-rest-proxy
  [8] scylla-cdc-ksqldb
  [9] scylla-cdc-kafdrop / scylla-cdc-control-center

  NETWORKING RULES
  ────────────────
  • All containers share scylla-cdc-bridge
  • Docker embedded DNS resolves hostnames within the network
  • ScyllaDB nodes communicate over gossip (:7000) and CQL (:9042)
  • Kafka Connect reaches ScyllaDB via scylla-cdc-quickstart:9042
  • Kafka Connect reaches broker via scylla-cdc-broker:9092
  • Host reaches all services via localhost:1xxxx port mappings
  • /etc/hosts is updated with all container IPs for host-side hostname resolution
